## About Xenode

Xenode is an open-source initiative focused on building privacy-first software for individuals, developers, and organizations.

Our goal is to create a modern ecosystem of applications that respect user privacy, give users full ownership over their data, and remain transparent through open-source development.

We believe that software should work for people—not business models built around collecting user data.

Xenode is currently being built and maintained by a solo developer with a long-term vision of creating a complete suite of privacy-focused applications and services.

This is only the beginning.

---

## Vision

The long-term vision for Xenode is to build an ecosystem of interconnected applications that provide users with powerful alternatives to traditional cloud services while preserving privacy and transparency.

Planned products include:

- End-to-end encrypted cloud storage
- Privacy-focused email services
- Local AI-powered applications
- Secure collaboration tools
- Productivity applications
- Developer-focused services
- Cross-platform desktop and mobile applications

Every product developed under Xenode will follow the same core principles:

- Privacy by default
- Open source whenever possible
- User ownership of data
- Transparency in development
- Community-driven innovation

## Current Focus

Xenode is currently focused on building a privacy-first cloud platform featuring:

- End-to-end encrypted storage
- Secure file management
- Privacy-preserving file sharing
- Local AI integrations
- Cross-platform support
- Zero-knowledge architecture

## Open Source Philosophy

Open source is at the heart of Xenode.

We believe that users should be able to inspect, audit, contribute to, and improve the software they depend on.

By developing in the open, we aim to build trust, encourage collaboration, and create software that benefits everyone.

Community feedback, discussions, issue reports, and contributions are essential to shaping the future of Xenode.

## Contributing

1. Fork the repository.
2. Create a feature branch.

```bash
git checkout -b feature/amazing-feature
```

3. Commit your changes.

```bash
git commit -m "feat: add amazing feature"
```

4. Push your branch.

```bash
git push origin feature/amazing-feature
```

5. Open a Pull Request.

Please make sure to follow the project's coding standards and contribution guidelines.
